Инкапсуляция механизм, который связывает вместе код и обрабатываемые им данные, а также защищает их от внешнего вмешательства и неправильного использования. 
Защитная оболочка, которая предотвращает произвольный доступ к коду и данным из другого кода, определенного вне оболочки.

Наследование - процесс, посредством которого один объект приобретает свойства другого объекта. Оно важно, т.к. поддерживает концепцию иерархической классификации

Полиморфизм в программировании – это свойство, позволяющее использовать одну и ту же сущность (например, метод или объект) 
для работы с объектами разных типов, при этом каждая сущность может иметь свою специфическую реализацию. Проще говоря, 
это возможность объектов разных классов выполнять одно и то же действие по-своему, что делает код более гибким, читаемым и расширяемым.

В Java весь код должен находиться внутри класса. 
Имя главного класса должно совпадать с именем файла, содержащего программу.

Ключевое слово static
заставляет член класса применяться независимо от любого объекта данного класса.

  При объявлении объектов такого класса копия статической переменной не создается.

  могут напрямую вызывать только другие статические методы своего класса
  могут напрямую получать доступ только к статическим переменным своего класса
  никоим образом не могут ссылаться на this или super.

блок static

Пакеты и интерфейсы

  Пакет является как механизмом именования, так и механизмом управления видимостью
    можете определять классы внутри пакета, которые не доступны коду вне пакета
    члены класса, которые видны только другим членам классов в том же пакете
      
  package пакет;
  
  в Java для хранения пакетов применяются каталоги файловой системы
  файлы . class для любых классов, которые объявляются как часть mypackage, должны храниться в каталоге с именем mypackage
  оператор package может находиться в нескольких файлах, он указывает, к какому пакету принадлежат классы, определенные в файле
  
  Допускается создавать иерархию пакетов
  package пакетl[.пакет2[.пакет3]]; 
  
  Поиск пакетов
    относительно текущего каталога 
    переменная среды CLASSPATH
    применить параметр -classpath при запуске java и javac
      
  Доступ к членам классов
  
                                  | private | без модификатора | protected | public 
    Тот же класс                  |    +    |         +        |     +     |    +    
    Подкласс из того же пакета    |    -    |         +        |     +     |    +    
    Не подкласс из того же пакета |    -    |         +        |     +     |    +    
    Подкласс из другого пакета    |    -    |         -        |     +     |    +    
    Не подкласс из другого пакета |    -    |         -        |     -     |    +

    import пакет1[.пакет2].( имя-класса | * ) ;

    то компилятор никак не будет реагировать, пока вы не попытаетесь использовать один из классов. В таком случае возникнет ошибка на этапе компиляции и

Интерфейсы
  Интерфейсы синтаксически похожи на классы, но в них отсутствуют переменные экземпляра и их методы МОГУТ объявляются без тела.
  Интерфейсы можно использовать для импортирования общих констант в несколько классов
    Когда такой интерфейс реализуется классом, имена всех этих переменных будут находиться в области видимости как константы.
    
  интерфейс может быть реализован любым количеством классов
  один класс может реализовывать любое количество интерфейсов
  
  Объявление 
  
    доступ interface имя {
      возвращаемый-тип имя-методаl ( список-параметров );
      возвращаемый-тип имя-метода2 ( список-параметров);
      тип финальное-имя-переменной1 = значение;
      тип финальное-имя-переменной2 = значение;
      // ...
      возвращаемый- тип имя-методаN ( список-параметров );
      тип финальное-имя-переменнойN = значение ;
    }
    
    Если модификатор доступа отсутствует, тогда интерфейс будет доступным только другим элементам пакета, в котором он объявлен. 
    Если puЫic, то его может использовать код вне пакета, где он объявлен. В таком случае интерфейс должен быть единственным открытым интерфейсом, объявленным в файле, а
    файл должен иметь то же имя, что и интерфейс. 
    
  Реализация интерфейсов 
  
    class имя-класса [extends суперкласс] [ implements интерфейс [, интерфейс ... ] ] {
      // тело класса
    }
    
    При реализации метода интерфейса он должен быть объявлен как public. 
    
  Доступ к реализациям через ссылки на интерфейсы
    
    Вы можете объявлять переменные как ссылки на объекты, в которых применяется тип интерфейса, а не тип класса. 
    С помощью переменной подобного рода можно ссылаться на любой экземпляр любого класса, реализующего объявленный интерфейс. 
    
  Частичные реализации
    Если класс включает интерфейс, но не полностью реализует методы, требуемые этим интерфейсом, то такой класс должен быть объявлен abstract
    
  Вложенные интерфейсы
    Интерфейс может быть объявлен членом класса или друrого и нтерфейса. Такой интерфейс называется членом-интерфейсом или вложенным интерфейсом. 
    Вложенный интерфейс может быть объявлен как public, private или protected. Когда вложенный интерфейс применяется
    за пределами своей области видимости, то он должен быть уточнен именем класса или интерфейса, членом которого является.
  
  Расширение интерфейсов
  
    // Один интерфейс может расширять другой .

    }
    // Интерфейс В теперь включает meth1 () и meth2 () - он добавляет methЗ ( ) .
    interface В extends А {
      void methЗ () ;
    }

Многопоточное программирование 

  два типа многозадачности
    
    на основе процессов
      процесс - это программа, которая выполняется
      многозадачность на основе процессов является функциональным средством, которое позволяет вашему компьютеру запускать две или большее число программ параллельно
      программа представляет собой наименьшую единицу кода, которая может координироваться планировщиком
    
    на основе потоков
      поток является наименьшей единицей координируемого кода
      одна программа может выполнять две или более задач одновременно
    
  Процессы - это тяжеловесные задачи, требующие отдельного адресного пространства, дорогое переключение между процессами
  Многозадачные потоки требуют меньше накладных расходов, совместно используют одно и то же адресное пространство
    
  Многопоточность позволяет сократить время простоя (ожидание данных по сети, чтения диска и т.д.) 
    
  Поток может
    выполняться
    быть готовым к запуску, как только получит время ЦП
    быть приостановлен
    Выполнение приостановленного потока затем можно возобновить
    может быть заблокирован при ожидании ресурса
    работа потока может быть прекращена
    
  Приоритеты потоков 
    целые числа, определяющие относительный приоритет одного потока над другим
    
    приоритет потока применяется для принятия решения о том, когда перек лючаться с одного работающего потока на другой
    переключением контекста
      Поток может добровольно передать управление
        Это происходит при явной уступке очереди, приостановке или блокировке. В таком сценарии проверяются все остальные потоки, 
        и поток с наивысшим приоритетом, готовый к выполнению, получает ЦП. 
      Поток может быть вытеснен потоком с более высоким приоритетом.
        В таком случае поток с более низким приоритетом, который не уступает ЦП, просто вытесняется - независимо от того, 
        что он делает - потоком с более высоким приоритетом. По сути, как только поток с более высоким приоритетом желает запуститься, он это делает
      
      Проблемы с переносимостью могут возникать из-за различий в том, как ОС переключают контекст для потоков с одинаковым приоритетом.

        boolean valueSet = false;

        synchronized int get() {
            while (!valueSet)
                try {
                    wait();
                } catch (InterruptedException e) {
                    System.out.println("Пepexвaчeнo исключение InterruptedException") ;
                }

            System.out.println ("Пoлyчeнo: " + n) ;
            valueSet = false;
            notify();
            return n;
        }

        synchronized void put (int n) {
            while (valueSet)
                try {
                    wait();
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    System.out.println("Пepexвaчeнo исключение InterruptedException") ;
                }

            this.n = n;
            valueSet = true;
            System.out.println ("Oтnpaвлeнo: " + n);
            notify();
        }
    }

  политики хранения 
    устанавливает момент, когда аннотация отбрасывается
    
    java . lang . annotation
    RetentionPolicy - 
    SOURCE - удерживается только в файле исходноrо кода и на этапе компиляции отбрасывается
    CLASS - на этапе компиляции сохраняется в файле . clas s. Однако она не будет доступной через машину JVM во время выполнения
    RUNTIME - на этапе компиляции сохраняется в файле . class и доступна через машину JVM во время выполнения
    
    Политика хранения для аннотации у казывается с применением одной из встроенных аннотаций Java: @Retention
    
    @Retention (retention-policy) 
      в арrументе retention-policy должна находиться одна из ранее обсуждавшихся констант перечисления, по умолчанию CLASS

          BiFunction<Integer, Integer, Integer> arithmeticOperation = getOperation(operation);
          //Попробуйте изменить аргументы
          int arg1 = 7;
          int arg2 = 3;

          //вызываем полученную лямбда-функцию с заданными аргументами

          System.out.println(result);

      private static BiFunction<Integer, Integer, Integer> getOperation(String sign) {
          switch (sign) {
              case "+": return (value1, value2) -> value1 + value2;
              case "-": return (value1, value2) -> value1 - value2;
              case "*": return (value1, value2) -> value1 * value2;
              case "/": return (value1, value2) -> value1 / value2;
              default:
                  throw new IllegalArgumentException("Неизвестная операция");
          }

      }

Stream
  на основе входного списка объектов создаётся СТРИМ — специальная последовательность данных и операций, которые будут с ними выполняться
  Для реализации таких операций, как трансформация и фильтрация последовательности объектов

  Stream API — набор классов Java для работы со стримами

  Виды операций
    Промежуточные (intermediate) — задают правила изменения стрима и возвращают трансформированный поток исходных данных. 
    Терминальные (terminal) — к ним относятся конечные операции, запускает выполнение всех промежуточных операций, 
          выполняет своё целевое действие и завершает работу над стримом. Она может вызываться у стрима только один раз.
    
  stream()
    методы
      map - трансформация,  применяет к каждому элементу стрима некоторое преобразование 
            В результате изначальный элемент стрима заменяется результатом преобразования.
      filter - фильтрация, проверяет каждый элемент стрима на соответствие условию 
              и исключает из стрима все элементы, не соответствующие ему
      peek - позволяет посмотреть на элементы стрима в процессе обработки, не изменяя их
      collect - преобразование стрима в тип 
    
    List<String> result = paths
                .stream()
                .filter(filePath -> !filePath.endsWith(".tmp"))
                .map(filePath -> Paths.get(filePath).getFileName().toString())
                .peek(processedPath -> System.out.println(processedPath))
                .peek(System.out::println)
                .collect(Collectors.toList());
    
Optional<T>
  это специальный класс-обёртка, объект которого может содержать значение класса T либо не содержать ничего
  например при запросе из БД не нашлось данных, вместо того чтобы возвращать null и обрабатывать NullPointerException используется данный класс
  Использование обёртки Optional информирует вызывающий код, что нужного значения может не быть и эту ситуацию нужно отдельно обработать.
    
  методы

    Optional.empty();

    orElseThrow - в качестве аргумента принимает лямбду, возвращающую исключение, которое нужно выбросить, если Optional пуст
    orElse - позволяет задать значение по умолчанию. В качестве аргумента он принимает значение того же типа, что и значение внутри Optional.
    orElseGet в качестве аргумента принимает лямбду типа Supplier. Этот метод нужно использовать, когда вычисление значения по умолчанию является дорогой операцией 
              и его выполнение нужно отложить до того момента, пока не будет точно известно, что Optional пуст.
    ifPresent Позволяет выполнить действие со значением внутри Optional. В качестве аргумента данный метод принимает лямбду типа Consumer. 
      .ifPresent(System.out::println)
    ifPresentOrElse В качестве аргументов принимает две лямбды типа Consumer и Runnable.

  @BeforeAll // static void  один раз до запуска всех тестов
  @AfterEach // static void  каждый раз после окончания каждого теста
  @AfterAll // void один раз после окончания всех тестов

public class DiscountCalculatorTest {

    DiscountCalculator discountCalculator = new DiscountCalculator();

    @Test
    public void shouldGiveNoDiscountForValue999() {
        // Подготовка
        int buySum = 999;
        int expectedSum = 999;

        // Исполнение
        int resultSum = discountCalculator.sumAfterDiscount(buySum);

        // Проверка
        // assertEquals(Object expected, Object actual, [String message])
        Assertions.assertEquals(expectedSum, resultSum);
    }
}

class DiscountCalculator {

    public int sumAfterDiscount(int sum) {
        if (sum < 1000) {
            return sum;
        } else {
            return (int) (sum * 0.99);
        }
    }
