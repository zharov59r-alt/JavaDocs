  Pointcut - выражение где должен быть применен Advice
    
    синтаксис выражения
    execution(modifiers? return-type declaring-type?method-name(parameters) throws?)
        ? - необязательный
        declaring-type - <package>.<class>.method-name
        *
          method-name - любые символы
          return-type - любой возвращаемый тип
          parameters любой параметр, 1 * = 1 параметр
        .. - parameters любое количество любых параметров
  
    Объявление pointcut
      @Pointcut("execution(* getBook(..))")
      public void pointcut_name() {}
      
      @Before("pointcut_name()")
      public void beforeLoggingAdvice() {System.out.println("Before Logging");}

  @Before - метод, выполнение до основного метода
  @AfterReturning - метод, выполнение после нормального окончания основного метода, НО до присвоения результата этого метода
    Перехват результата
      public List<Student> getStudents(){}
      @AfterReturning(pointcut = "execution(* getStudents())", returning = "students")
      public void afterReturningGetStudentsLoggongAspect(List<Student> students) {}
  
  @AfterThrowing - метод, выполнение после основного метода, только если сработал exception
    Инфа по ошибке
      @AfterThrowing(pointcut = "execution(* getStudents())", throwing = "exception")
      public void afterThrowingGetStudentsLoggongAspect(Throwable exception) {
    
  @After - метод, выполнение после основного метода независимо от успешности
    нельзя получить доступ к исключению и к результату
    
  @Around - метод, выполняется до и после основного метода
    target метод вызывается вручную в методе обработки
    позволяет получить доступ к исключению и к результату
    
      @Around("execution(* second.Book.*(..))")
      public Object aroundAllMethods(ProceedingJoinPoint proceedingJoinPoint) throws Throwable {
        long startTime = System.currentTimeMillis();
        Object targetMethodResult = null;
        try {
            targetMethodResult = proceedingJoinPoint.proceed();
        } catch (Exception e) {
            System.out.println("aroundAllMethods Exception: " + e.getMessage());
            
            targetMethodResult = "default result";
            ИЛИ
            throw e; // проброс исключения дальше
        }
        System.out.println("aroundAllMethods: " + (System.currentTimeMillis() - startTime) + "ms");
        return targetMethodResult;
    }

      Eager - связанные сущности загружаются вместе с основной (когда мы селектим employee оно генерит join department)
      Lazy - связанные сущности НЕ загружаются вместе с основной, загружаются при обращении к ним

  CriteriaBuilder criteriaBuilder = entityManager.getCriteriaBuilder();
  CriteriaQuery<Student> criteriaQuery = criteriaBuilder.createQuery(Student.class);
  Root<Student> root = criteriaQuery.from(Student.class);
  // объявляем именованный параметр
  ParameterExpression<String> nameParam = criteriaBuilder.parameter(String.class, "name");
  // описываем условие добавляемое в where
  Predicate namePredicate = criteriaBuilder.equal(root.get("name"), nameParam);
  // добавляем условие в where
  criteriaQuery.where(namePredicate);
  criteriaQuery.select(root);
  TypedQuery<Student> query = entityManager.createQuery(criteriaQuery);
  query.setParameter("name", "1");
      
      
  // выборка нескольких полей
  CriteriaBuilder criteriaBuilder = entityManager.getCriteriaBuilder();
  CriteriaQuery<Tuple> criteriaQuery = criteriaBuilder.createTupleQuery();
  Root<Student> root = criteriaQuery.from(Student.class);
  Path<String> namePath = root.get("name");
  Path<String> surnamePath = root.get("surname");
  CompoundSelection<Tuple> select = criteriaBuilder.tuple(namePath,  surnamePath);
  criteriaQuery.select(select);
  TypedQuery<Tuple> query = entityManager.createQuery(criteriaQuery);
      
  // join
  CriteriaBuilder criteriaBuilder = entityManager.getCriteriaBuilder();
  CriteriaQuery<Tuple> criteriaQuery = criteriaBuilder.createTupleQuery();
  Root<University> rootUniversity = criteriaQuery.from(University.class);
  Join<University, Student> joinStudent = rootUniversity.join("students");
  Path<String> nameUniversityPath = rootUniversity.get("name");
  Path<String> nameStudentPath = joinStudent.get("name");
  CompoundSelection<Tuple> select = criteriaBuilder.tuple(nameUniversityPath,  nameStudentPath);
  criteriaQuery.select(select);
  TypedQuery<Tuple> query = entityManager.createQuery(criteriaQuery);

  создается при создании экземпляра EntityManager
  посредник между приложением и БД
  экземпляр EntityManager в Spring Boot создается при необходимости работы с базой данных, обычно внутри транзакции или при запросе компонента 
  экземпляр EntityManagerFactory - создается на уровне приложения
    
    Управление сущностями: Интерфейс EntityManager используется для управления этим контекстом. 
                          Все действия (persist, merge, remove, find) происходят внутри него
    Кэш первого уровня: При запросе сущности EntityManager сначала ищет её в контексте, уменьшая количество запросов к БД
    Синхронизация (Flush): Изменения, внесенные в сущности, не отправляются в базу данных немедленно. 
                          Сброс (flush) происходит автоматически при завершении транзакции или принудительно через flush()
    Отслеживание изменений (Automatic Dirty Checking):
                          При помещении объекта в контекст создается его снапшот (исключаеются поля @Transient)
                          при выполнении flush и commit SQL генерится только для измененных объектов
                          
                            
                            
    
  Entity States
                                                                                                                                       
  
    Transient (New) - проставляется при создании экземпляра entity
    Persistent (Managed) - Entity находится в контексте
                          проставляется при вызове метода persist() на Transient экземпляре
                          при получении экземпляра через find()
                          
                          добавляет Entity в контекст
    Detached - проставляется при закрытии EntityManager или вызове detach()
                экзепляр убирается из контекста и не будет синхронизирован с БД
    Removed - проставляется при вызове remove(), объект будет удален из БД при синхронизации

  Метод 
    void persist(Object var1) - делает новый объект сущности управляемым (Managed) и персистентным, 
                                добавляя его в контекст сохранения. Он регистрирует объект для последующей 
                                вставки в базу данных (SQL INSERT) при фиксации транзакции (commit) или при flush
    boolean contains(Object var1) - проверяет находится ли Entity в Persistence context
    void flush() - принудительно синхронизирует состояние контекста персистентности с БД - выполняет накопленные SQL
    void detach(Object var1) - изменяет статус на Detached
                              можно использовать для экономии памяти и ускорения фиксации, 
                              т.к. придется меньше объектов проверять на необходимость синхронизации
    void refresh(Object var1) - обновляет состояние управляемой сущности, перезаписывая её текущие значения данными из базы данных
    void clear() - очищает контекст персистентности, отсоединяя (detaching) все загруженные сущности от менеджера
    <T> T merge(T var1) - переносит состояние отсоединенного (detached) объекта в персистентный контекст (Persistence Context), обновляя данные в БД
                          делает select по ID, копирует в экземпляр изменения из переданного объекта и возвращает управляемую (managed) >>> КОПИЮ <<< 
                          При этом исходный переданный объект остается отсоединенным.
                          при синхронизации если объекта в БД не нашлось, то выполнится insert, если нашелся, то update всех полей

Наследование Entity

  Single Table

    @DiscriminatorColumn(name = "emp_type")
    @Inheritance(strategy = InheritanceType.SINGLE_TABLE)
    public abstract class Employee

    public class Teacher extends Employee
    
    будет ожидать таблицу Employee с полем emp_type text в которую будет инсертить "Teacher"

  Joined
  
    @Inheritance(strategy = InheritanceType.JOINED)
    public abstract class Employee

    public class Teacher extends Employee
  
    будет ожидать таблицу Employee и Teacher дополненную полем id, по сути это как OneToOne

    При этом при запросе Employee будет left join Teacher, а если наследников больше, то все будут left join...
  
  Table per class
  
    @Inheritance(strategy = InheritanceType.TABLE_PER_CLASS)
    public abstract class Employee

    public class Teacher extends Employee
    
    будет ожидать таблицу Teacher дополненную всеми полями из Employee
    При этом при запросе Employee будет union всех дочерних

        class LocalDateAdapter extends TypeAdapter<LocalDate> {
            private static final DateTimeFormatter dtf = DateTimeFormatter.ofPattern("dd.MM.yyyy");

            @Override
            public void write(final JsonWriter jsonWriter, final LocalDate localDate) throws IOException {
                jsonWriter.value(localDate.format(dtf));
            }

            @Override
            public LocalDate read(final JsonReader jsonReader) throws IOException {
                return LocalDate.parse(jsonReader.nextString(), dtf);
            }
        } 
        
        GsonBuilder gsonBuilder = new GsonBuilder();
        gsonBuilder.registerTypeAdapter(LocalDate.class, new LocalDateAdapter());
        Gson gson = gsonBuilder.create();

HTTP client

  java.net.URI — описывает адрес нужного ресурса.
  
  java.net.http
    HttpRequest — описывает HTTP-запрос к серверу. Экземпляры этого класса создаёт разработчик программы HTTP-клиента.
    HttpResponse — описывает HTTP-ответ сервера. Экземпляры этого класса возвращаются в ответ на отправленные программой запросы.
    HttpClient — реализует коммуникацию с сервером по протоколу HTTP. Отправляет запросы, 
        описанные разработчиком в виде экземпляров HttpRequest, а также получает ответы и преобразует их в экземпляры HttpResponse.

  // создаём экземпляр URI, содержащий адрес нужного ресурса
  URI uri = URI.create("https://ru.wikipedia.org/wiki/Список_кодов_состояния_HTTP");

  // получаем экземпляр класса-строителя
  HttpRequest.Builder requestBuilder = HttpRequest.newBuilder();

  // создаём объект, описывающий HTTP-запрос
  HttpRequest request = requestBuilder
          .GET()    // указываем HTTP-метод запроса
          .uri(uri) // указываем адрес ресурса
          .version(HttpClient.Version.HTTP_1_1) // указываем версию протокола HTTP
          .header("Accept", "text/html") // указываем заголовок Accept
          .build(); // заканчиваем настройку и создаём ("строим") HTTP-запрос 

  HttpClient client = HttpClient.newBuilder() // получаем экземпляр билдера
        .connectTimeout(Duration.ofSeconds(5)) // указываем максимальное время ожидания соединения с сервером
        .build(); // заканчиваем настройку и создаём ("строим") HTTP-клиент 

  или с настройками по умолчанию
  HttpClient client = HttpClient.newHttpClient(); 

    метод
      <T> HttpResponse<T> send(
        HttpRequest request, 
        HttpResponse.BodyHandler<T> responseBodyHandler)
              интерфейс, который помогает HTTP-клиенту преобразовать тело ответа из набора байтов в Java-объект. 
              В Java имеются готовые реализации этого интерфейса. 
              Они доступны в классе HttpResponse.BodyHandlers. Например, HttpResponse.BodyHandlers.ofString() возвращает обработчик, 
              позволяющий преобразовать тело ответа из набора байтов в строку.
        throws IOException, InterruptedException;

  HttpResponse.BodyHandler<String> handler = HttpResponse.BodyHandlers.ofString();
  HttpResponse<String> response = client.send(request, handler);
