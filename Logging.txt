Компоненты фреймворка логирования
  Logger — специальный класс, который преобразует логируемое событие во внутренний объект и передаёт его дальше другим компонентам фреймворка.
  Appender — интерфейс, который отвечает за добавление объекта, полученного от логера, непосредственно в лог
  Layout — интерфейс, который перед выводом записи из лога в консоль или на печать приводит её к определённому формату

Logger
  В качестве имён для логеров принято использовать полное название класса, включающее название пакетов, в которые этот класс входит
    
  import org.slf4j.Logger;
  import org.slf4j.LoggerFactory;

  class MyClass {
    private static final Logger log = LoggerFactory.getLogger(MyClass.class);
    ИЛИ private static final Logger log = LoggerFactory.getLogger("MyClass");
    }
  
Appenders
  
  ConsoleAppender — выводит записи в консоль.
  FileAppender — сохраняет записи в файл.
  RollerFileAppender — аналог FileAppender с функцией автосоздания нового лог файла при достижении размера
  
Layout
  
  PatternLayout - класс позволяет очень гибко настраивать формат строки для вывода записи
    пример **%d{HH:mm:ss}: %message%n** 
    
Уровни важности

  TRACE — для трассировочных сообщений (выбор ветки в условии if — else, значения параметров внутри циклов, каждый шаг алгоритма и так далее)
  DEBUG — для отладочных сообщений, которые важны во время анализа некорректного поведения программы
  INFO
  WARN  — например если было сгенерировано исключение или обнаружена проблема в ходе выполнения алгоритма
  ERROR — для сообщений о критических ошибках, требующих оперативного вмешательства, без которого программа не сможет продолжить работу
  

Настрофка
  Чтобы пользоваться параметрами Logback придется сделать приведение логгера

  private final static Logger log = (ch.qos.logback.classic.Logger) LoggerFactory.getLogger(Practicum.class);
  
   log.setLevel(Level.WARN); // задает минимальный уровень для вывода в журнал
